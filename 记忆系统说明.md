# My Neuro 记忆系统说明

My Neuro 的记忆系统不是一个单独的小插件，而是一套由多个插件协同工作的“记忆中枢”。它让 AI 不只是当场聊天，而是能把重要信息沉淀下来，在之后的对话里重新想起来、判断轻重、形成用户画像，并用 WebUI 管理这些记忆。

你可以把它理解成 AI 的第二大脑：

- MemOS 长期记忆：负责保存、搜索、整理长期记忆。
- 核心记忆注入：负责把最关键、最稳定的信息放进 AI 的“长期注意力”里。
- 用户画像：负责从长期对话里沉淀用户偏好、习惯、雷区和关系信息。
- AI 日志：负责把每天的对话整理成日志，让 AI 有连续的生活记录。
- 晨昏之线：负责时间、时区、节假日和整点问候，让 AI 知道“现在是什么时候”。

这些插件组合在一起后，AI 会更像一个能长期陪伴你的角色：它会记得你说过的重要事，会逐渐理解你的偏好，也能在合适的时候自然提起过去的内容。

## 快速启动

第一次使用时，建议按这个顺序来：

1. 双击根目录的 `4.MEMOS-API.bat`
   - 这是记忆系统后端。
   - 看到窗口持续运行即可，不要关掉。
   - 默认地址是 `http://127.0.0.1:8003`。

2. 可选：双击根目录的 `5.MEMOS-WebUI.bat`
   - 这是记忆管理控制台。
   - 浏览器会打开 `http://127.0.0.1:8004`。
   - 你可以在里面查看、搜索、修正、管理记忆。

3. 正常启动 My Neuro 主程序
   - 记忆相关插件默认开启。
   - 插件会自动连接 `http://127.0.0.1:8003`。
   - 如果插件里没有单独填写 API 和模型配置，会优先回退使用主对话模型。

日常使用时，最简单的顺序就是：

```text
先开 4.MEMOS-API.bat
再开 My Neuro 主程序
需要管理记忆时，再开 5.MEMOS-WebUI.bat
```

## 默认开启的记忆插件

这套记忆系统默认包含下面几个核心插件：

| 插件 | 作用 |
| --- | --- |
| MemOS 长期记忆 | 自动保存对话、搜索历史、导入文档/网页、管理图片记忆和工具记录 |
| AI 日志 | 按天整理 AI 的观察日志，也可以形成更长期的回忆记录 |
| 核心记忆注入 | 把“永远不该忘”的信息稳定注入，例如名字、关系、重要设定 |
| 用户画像 | 从多次对话里保守沉淀偏好、习惯、禁忌和长期特征 |
| 晨昏之线 | 注入时间、时段、节假日、调休信息，并支持整点问候 |

这几个插件各自负责一层记忆：有的管“资料库”，有的管“人格理解”，有的管“核心事实”，有的管“时间感”。组合起来之后，AI 不只是能搜索过去，而是能把过去变成当前对话的一部分。

## 专业版能力总览

如果说普通聊天记录只是“把话存下来”，那么 My Neuro 的记忆系统更接近一个轻量级 Memory Operating System。它把对话、偏好、图片、工具调用、文档知识和用户画像统一放进一套可检索、可演化、可修正的长期记忆管线里。

它的完整流程大概是这样：

```text
对话输入
→ 重要信息抽取
→ 语义向量化
→ 长期记忆入库
→ BM25 关键词索引
→ 知识图谱实体链接
→ 多路召回
→ CrossEncoder 精排
→ 记忆注入到下一轮对话
→ 日志、画像、核心记忆继续沉淀
```

这听起来复杂，但用户实际只需要正常聊天。复杂的部分都在后台自动完成。

### MemOS 的核心技术层

MemOS 是整套记忆系统的底座，它不是简单的 txt 记事本，而是一套多层检索和记忆治理系统：

| 技术层 | 作用 | 用户能感受到什么 |
| --- | --- | --- |
| Qdrant 向量记忆库 | 把每条记忆变成高维语义向量，支持“意思相近也能搜到” | 不用说完全一样的话，AI 也能想起相关内容 |
| BM25 关键词检索 | 建立传统关键词倒排索引，补足专有名词、文件名、ID、术语搜索 | 搜项目名、插件名、报错关键词更准 |
| NetworkX 知识图谱 | 抽取人物、项目、地点、概念之间的关系 | AI 不只记住单句话，还能理解“谁和什么有关” |
| CrossEncoder 精排 | 对召回候选进行二次相关性判断 | 相关记忆排得更靠前，减少“搜到了但用错了”的情况 |
| 记忆演化机制 | 根据访问次数、重要性、时间跨度做晋升、衰减、归档、合并 | 常用记忆更稳定，过期碎片会慢慢降权 |
| Function Call 工具层 | 让 AI 主动搜索、添加、修正、导入和查询记忆 | AI 能自己调用工具，不只是被动等待 |

### 记忆不是越多越好，而是要有治理

长期陪伴型 AI 最大的问题不是“能不能存”，而是“存多了以后还能不能用”。所以这套系统加入了几类治理策略：

- 重要性评分：每条记忆都有 importance，不重要的流水账不会占据太高优先级。
- 多路召回融合：向量相似度、关键词命中、图谱关系、时间新鲜度会一起影响结果。
- 记忆分层：WorkingMemory、LongTermMemory、UserMemory 分工不同，临时上下文和长期事实不会混成一锅。
- 频率与时间权重：经常被用到的记忆会更容易被召回，长期不复现的信息会逐步降低存在感。
- 合并与去重：相近记忆可以通过演化流程逐渐合并，减少重复碎片。
- 候选池制度：用户画像不会因为一句话立刻定型，而是先进入候选池，积累证据后再晋升。

换句话说，它不是“无限往本子里写”，而是在模拟一种更接近长期认知系统的记忆生命周期。

## MemOS 功能矩阵

MemOS 插件向 AI 暴露了一组记忆工具。新手不需要记工具名，但了解它们能帮助你知道系统有多强。

| 能力 | 工具/模块 | 能做什么 |
| --- | --- | --- |
| 深度记忆检索 | `memos_search_memory` | 按语义、关键词和图谱关系搜索历史记忆 |
| 手动添加记忆 | `memos_add_memory` | 把用户明确要求记住的内容写入长期记忆 |
| 记忆修正 | `memos_correct_memory` | 修改、补充、删除错误记忆，避免越记越偏 |
| 偏好查询 | `memos_get_preferences` | 查询用户喜欢什么、不喜欢什么、常见习惯 |
| 图片记忆 | `memos_upload_image`、`memos_search_images` | 保存图片及描述，并支持之后检索 |
| 截屏记忆 | `memos_save_screenshot` | 把关键画面保存成可搜索记忆 |
| 文件图片入库 | `memos_save_image_from_file` | 从本地图片文件生成图片记忆 |
| 工具使用记录 | `memos_record_tool_usage`、`memos_search_tool_usage` | 记录 AI 做过哪些操作，下次可以参考经验 |
| 知识库导入 | `memos_import_url`、`memos_import_document` | 导入网页、txt、pdf、md 等资料，变成可检索知识 |

举个更具体的例子：

```text
你说：以后公开提交前要检查 key、本地路径、日志、私密提示词。
MemOS：把这句话抽取成长期记忆。
用户画像：把“重视隐私和提交安全”作为候选观察。
核心记忆注入：可以把这类硬规则放进稳定背景。
AI 日志：当天会记录“用户强调仓库公开前要做隐私扫描”。
下次你说：帮我发 PR。
AI：会更自然地先检查敏感信息，而不是每次都等你提醒。
```

这就是多插件协同的意义：不是某一个插件单点变强，而是形成一套“长期上下文、行为偏好、核心规则、时间感知、操作复盘”共同工作的记忆网络。

## 它到底能做什么

### 1. 自动回忆相关内容

当你提到一个话题时，MemOS 会在后台搜索相关记忆，把最相关的内容注入给 AI。

例如你以前说过：

```text
我最近在做 Live2D 项目，最头疼的是动作切换和表情绑定。
```

之后你问：

```text
上次那个模型切换问题怎么继续排查？
```

AI 就有机会想起之前的上下文，而不是像第一次听说一样从零开始。

### 2. 自动沉淀长期信息

普通聊天记录很容易变成一大堆流水账。记忆系统会尽量把对话压缩成更有价值的长期信息，例如：

```text
用户正在维护一个 Live2D/VRM 相关项目。
用户倾向于先看计划，再让助手施工。
用户希望公开仓库不要包含敏感信息、提示词和私密数据。
```

这类信息比完整聊天记录更轻、更稳定，也更适合长期使用。

### 3. 形成用户画像

用户画像插件不会因为一句话就随便下结论。它会把近期观察先放进候选池，等多次出现、跨天出现，或者用户明确表达后，才晋升为正式画像。

例如：

```text
候选观察：用户喜欢先确认方案。
多次证据：用户多次要求“先别更新，先给计划”。
正式画像：用户偏好先看计划，再执行代码或仓库操作。
```

这种设计更稳，不容易因为一次玩笑或临时情绪误判用户。

### 4. 核心记忆长期稳定注入

有些信息不适合每次都靠搜索，例如称呼、重要身份、长期设定、绝对不能忘的规则。核心记忆注入插件会把这些内容作为稳定背景提供给 AI。

适合放进核心记忆的例子：

```text
用户希望回答尽量浅显易懂。
用户不希望公开仓库上传 API Key、个人日志和私密提示词。
用户喜欢先看到计划，再进行较大的修改。
```

### 5. AI 日志与长期复盘

AI 日志插件会把一天里的重要对话整理成更像“观察日志”的内容。它适合用来做长期陪伴、角色连续性、月度回忆，也适合回看某段时间发生了什么。

它不是简单复制聊天记录，而是更像把一天的经历整理成摘要：

```text
今天用户主要在整理记忆系统 PR，强调默认开启记忆插件，并要求保护隐私数据。
```

### 6. 时间与环境感知

晨昏之线会告诉 AI 当前时间、星期、时段、节假日或调休状态。这样 AI 的回应会更贴近现实场景。

例如：

```text
早上：自然说早安，语气更清醒轻快。
中午：可以提醒吃饭和休息。
深夜：可以温柔提醒早点休息。
节假日：知道今天不是普通工作日。
```

## WebUI 能做什么

`5.MEMOS-WebUI.bat` 打开的管理界面可以理解为“记忆控制台”。它适合做这些事：

- 查看当前保存了哪些记忆。
- 搜索某个话题的历史记忆。
- 手动添加重要记忆。
- 修改、补充或删除错误记忆。
- 查看知识图谱和实体关系。
- 导入文档、网页等外部资料。
- 检查图片记忆、工具使用记录等扩展数据。

如果你是新手，不需要一开始就理解所有功能。最常用的就是：搜索记忆、修正记忆、添加一条重要记忆。

## 关于精排模型

MemOS 支持 CrossEncoder 精排模型，它会在“先粗略搜出一批记忆”之后，再用更强的模型重新排序，让最相关的记忆排到前面。默认模型是 `BAAI/bge-reranker-v2-m3`，体积较大，所以默认启动不会直接下载，避免新手第一次启动时被大模型下载卡住。

如果你想启用它，只需要在 MemOS 插件配置里打开：

```text
启用 CrossEncoder 精排
```

“自动下载精排模型”默认已经开启。也就是说，只要你打开精排功能，第一次启动后端时，系统会把模型下载到项目根目录的 `full-hub/reranker-hub`。如果不启用精排，MemOS 仍然会使用向量检索、BM25 和知识图谱增强检索，只是少了一层更高级的二次排序。

## 推荐使用方式

### 新手模式

只做三件事：

```text
1. 启动 4.MEMOS-API.bat
2. 启动 My Neuro 主程序
3. 正常聊天
```

记忆系统会在后台自动工作。你不需要手动维护每一条记忆。

### 进阶模式

当你发现 AI 记错了，或者想让它牢牢记住一件事，可以打开 WebUI：

```text
1. 启动 5.MEMOS-WebUI.bat
2. 搜索相关记忆
3. 修改错误内容，或新增正确记忆
```

例如你可以手动添加：

```text
用户正在维护 my-neuro 项目，公开提交前必须检查是否包含 API Key、私密提示词、本地路径和运行日志。
```

这样之后 AI 在处理仓库任务时，就更容易自动注意到这些要求。

## 常见问题

### 记忆没有生效怎么办？

先检查 `4.MEMOS-API.bat` 的窗口是否还在运行。MemOS 后端关闭后，主程序里的记忆插件就无法检索长期记忆。

### WebUI 打不开怎么办？

先确认 `4.MEMOS-API.bat` 已经启动，再运行 `5.MEMOS-WebUI.bat`。WebUI 需要后端 API 正常运行。

### 我需要手动填写 API Key 吗？

通常不用。记忆插件的专用模型/API 配置如果留空，会尽量使用主对话模型。只有你想让记忆总结、画像分析使用另一个模型时，才需要单独配置。

### 记忆会不会越存越乱？

记忆系统不是单纯堆聊天记录。它会用检索、摘要、重要性、画像候选池、核心记忆等多层机制，把“流水账”和“长期有用的信息”分开处理。必要时你也可以在 WebUI 里手动修正。

### 我的隐私数据在哪里？

记忆数据主要保存在本机的 MemOS 数据目录和 AI 记录目录里。公开提交仓库时，不应该上传真实记忆库、日志、图片、数据库、API Key 或私密提示词。

## 一句话总结

这套记忆系统的目标，是让 My Neuro 从“每次重新认识你”升级成“能长期理解你”。它既有可搜索的长期记忆，也有稳定注入的核心事实，还有会逐步成熟的用户画像和每日日志。对新手来说，只要按顺序启动两个 BAT，就能让 AI 获得更连续、更聪明、更像长期伙伴的记忆能力。
